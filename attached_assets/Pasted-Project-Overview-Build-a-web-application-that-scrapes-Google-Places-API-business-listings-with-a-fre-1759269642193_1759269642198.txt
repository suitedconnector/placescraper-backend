Project Overview
Build a web application that scrapes Google Places API business listings with a freemium model. Users can search by location (state/city/multiple zip codes) and keywords, with tiered access levels.
MVP Phase 1 - Core Functionality (Build This First)
Focus on getting YOU testing ASAP with these essential features:
Authentication & User Management

Simple email registration (no email verification for MVP - add later)
Secure login system
User dashboard showing:

Google API key input field (encrypted storage using environment variables + database encryption)
API usage tracker (track calls made through our app)
Saved searches list



Search Interface (Single Page)
Location Filters:

State: Dropdown (all US states)
City: Text input (since cities vary by state)
Zip Codes: Text input accepting comma-separated values (e.g., "90210, 90211, 90212")

Business Filters:

Keywords: Text input (comma-separated)
Keyword Logic: Radio buttons ("Match ALL keywords" / "Match ANY keywords")
Google Business Category: Multi-select dropdown (populate from Google Places API types)
Results Limit: Number input (default 100, max 500)

Action Buttons:

"Search Now" button
"Save This Search" button (saves criteria for later reuse)

Results Display

Table View (default):

Columns: Business Name, Address, Phone, Website, Rating, Review Count, Photo URLs (clickable)
Sortable columns
Pagination (50 results per page)


Card View (toggle button):

Grid layout with business cards
Each card shows: Name, address, rating stars, phone, thumbnail photo


Progress Bar during API calls with estimated time remaining

Export (MVP - Free Tier Only)

CSV download button (all fields)
Include all business data: name, address, phone, website, rating, review count, hours, photo URLs, place_id, lat/lng

Error handling for invalid API keys with helpful messages

Technical Specifications
Recommended Tech Stack
Let Replit choose, but suggest:

Backend: Python (Flask) or Node.js (Express)
Database: SQLite for MVP (easy, no setup), upgrade to PostgreSQL later
Frontend: HTML/CSS/JavaScript with Bootstrap or Tailwind CSS
API Key Encryption: Use Fernet (Python) or crypto module (Node.js)

Security Requirements (CRITICAL)
API Key Storage Best Practice:

NEVER store API keys in plain text
Use AES-256 encryption with a master key stored in environment variables
For free tier: Store API key only in session (memory), cleared on logout
For paid tier: Encrypt API key before saving to database, decrypt only when making API calls
Implement rate limiting to prevent abuse (max 10 searches per hour for free tier)

Database Schema (Minimal for MVP)
Users table:
- id, email, password_hash, tier (free/pro), created_at, api_key_encrypted (NULL for free)

Saved_Searches table:
- id, user_id, search_name, state, city, zip_codes, keywords, keyword_logic, category, results_limit, created_at

App_Usage table:
- id, user_id, api_calls_made, search_date
Google API Calls Strategy

Use Places API Text Search for initial query
Fetch Place Details for each result (gets all data + photos)
Implement caching for repeated searches (1-hour cache)
Show user: "This search will use approximately X API calls"

User Experience Flow (MVP)

Dashboard → "Enter your Google API Key" (with link to get one)

Search form appears → Fill filters → Click "Search Now"
